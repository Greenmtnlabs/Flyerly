Gif Factory App - iOS full application

Gif Factory App is an iOS application that helps in creating the high quality animated GIF images with a lot of customization options. Its functionality includes the ability to create GIFs by combining the static images (such as .png, jpg etc.), convert the video to the animated sequence in accordance with the selected time range, edit and mix the existing GIFs, preview GIFs from the Photos Library and other useful features. It was designed to make the process of turning your favorite photos/videos into the GIFs quick and easy.

This App uses the CocoaPods dependency manager, so some parts of it (such as the VideoTrimmerView pod) can be easily integrated with your existing projects.

Features

App Features
  • Create GIFs from saved Photos and Videos or combine the existing GIFs into the new ones;
  • Rearrange the frames of your GIFs;
  • Control the animation parameters such as: dimensions, frame delay, loop count, content mode etc;
  • Preview GIFs from the Photos App Library;
  • Save the resulting GIFs to the App Documents folder or directly to the Photos App Library;
  • Share the GIF using any app extensions installed on your device (such as Facebook or Twitter);
  • Share your GIF files via email or Clipboard;
Programming related features
  • Project files are compatible with the Xcode version 7 and above;
  • It was written using the official release of the Swift programming language, version 2.2;
  • Lazy GIF loading to reduce the device memory usage;
  • Use Auto Layout and Size Classes to simplify building the UI for any iOS device;
  • Use Storyboards to allow you customize the interface of this App easily and quickly;
  • Landscape & portrait modes: specially coded to support any device orientation;
  • Fully-documented code with comments for the most important parts;
  • Website with Api Documentation in Apple style was generated for your convenience;
  • Can be easily translated into any language (all the displayable strings are provided through the NSLocalizedString context);
  • Support iOS 8 & 9, iPhone & iPad.
 
 

Project Structure

After extracting the downloaded GifFactoryApp.zip archive you will be presented with the following direcotry structure:

 

GifFactoryApp
  |-- api-docs
  |-- build
  |-- Documentation
  |-- GifFactoryApp
  |-- GifFactoryAppTests
  |-- Pods
  |-- Private Pods Sources
  |-- Resources

 

The root GIfFactoryApp (don't confuse it with the GifFactoryApp subfolder) folder containts the main project files and resources. This is the place where you can open the project  in Xcode using the GifFactoryApp.xcworkspace file, check out the documentation of the project API (located in the api-docs subfolder), find used in the project fonts and images (in the Resources subfolder), check out the source code of all installed pods (in the Pods subfolder) and edit the specifications of the project dependencies in the podfile (more info about the podfiles you can find at this link).

In the Documentation folder you will find the generated website containing the Gif Factory project documentation. Double-click an index.html to open it in your favorite browser. This is the first place your should check out if you feel that you don't understand how the basic things (such as the project compilation/installation) work.

The Private Pods Sources folder contains the sources of the privately developed pods. Currently it includes the VideoTrimmerView pod that helps the users to quickly select the video time range and convert it to the animated GIF. It provides the scalable timeline which can be configured depending on the video clip length. 

By default, the private pods are pre-installed for your convinience. However, if you want to extend its functinality according to your needs you should follow the next basic steps:

  • Create a private Spec repository on your server;
  • Add the private repo to your CocoaPods Installation;
  • Add the PrivatePodName.podspec to the private repo (for example, the podspec file of the VideoTrimmeView pod is called VideoTrimmerView.podspec);
  • Update the source reference in the podfile that is located in the GifFactoryApp folder (it should look like this: source 'URL_TO_YOUR_REPOSITORY'.

After completing the above steps you will be able to install the updated private pod by running the following command from the terminal:

cd GifFactoryApp
pod update

More info about the private pods you can find at the link.

Compilation/Installation

This project uses the CocoaPods dependency manager that’s why the main file you use to open the project in Xcode is GifFactoryApp.xcworkspace (NOT GifFactoryApp.xcodeproj !!). Generally, it can be useful to clean code before building using the Xcode Clean command from the Product menu.
 
To install and run the app on an iOS Simulator just select one of the available in the system simulators and press Command-R.
 
To build and run the app on your iOS Device you should be a member of the Apple Developer Program. If you don’t want to join the Apple Developer Program, you can still build and run your app on a Device using a valid Apple Id (which is free). But in this case the capabilities available to the app will be restricted. Check out this page to know more about the capabilities dependencies.
 
Use the Apple’s official guide to know how to configure Xcode with your Apple Id.
 
The general information about preparing and configuring an app to launch on devices is available at this link.
 
For your convenience here are the basic steps to getting the app onto your iOS device using your Apple Id:
 
  • Choose Xcode > Preferences;
  • At the top of the window, click Accounts;
  • In the lower-left corner, click the Add button;
  • Choose Add Apple Id from the pop-up menu;
  • In the dialog that appears, enter your Apple ID and password, and click Sign In;
  • In the project navigator, select the project and target to display the project editor;
  • Click General and choose your name from the Team pop-up menu;
  • Connect the device to your Mac and choose your device from the Scheme toolbar menu;
  • Below the Team pop-up menu, click Fix Issue.
  • Click the Run button.

 

AdMob

Currenlty the following ads from the AdMob SDK are integrated into this app:

  • Smart Banner Ads - these ads are displayed at the bottom of the main screen. The banner size is automatically updated in accordance with the device orientation. The border is added to the banner's top side to prevent AdMob policy violation (check out this link);
  • Interstitial Ads - these ads are presented each time the new GIF is saved;

Important!

During development and testing, use test ads. If you do need to render live ads before launch, avoid clicking on them.

If you click on live ads, your AdMob account may be suspended.


To configure ads in accordance with your AdMob account settings perform the next steps:

  • Replace the GoogleService-Info.plist with the file obtained from the Firebase Console;
  • Create ad unit IDs for the banner and intersitial using official tutorial;
  • Set the appropriate unit ids using the following properties (located in the GFMainScreenViewController.swift):

    • /// The AdMob banner ad unit id.
private let kBannerAdUnitId = "ca-app-pub-3940256099942544/2934735716"
/// The AdMob interstitial ad unit id.
private let kInterstitialAdUnitId = "ca-app-pub-3940256099942544/4411468910"

The more info about AdMob configuration you can find here.

Api Docs Generation

If you have extended the existing Api of this project and want to update the website that is located in the api-docs folder you must have the Jazzy tool installed on your system.

When the installation is done run the following commands using the Terminal:

cd GifFactoryApp/
jazzy

 

The jazzy command will use the .jazzy.yaml configuration file during the generation process. Feel free to edit it as you wish. In accordance with the Jazzy documentaion, for a detailed explanation and an exhaustive list of all available options, run this:

jazzy --help config.

Customisation

This App can be fully customized in accordance with your preferences. 

Global customization

The main customization place is located in GFGifFactoryAppDelegate.swift file, in application(_:, didFinishLaunchingWithOptions:) function. Here with the help of the appropriate  UIAppearance proxy object you can customized properties of the most UI elements used in the project. Instead of having to customise the appearance of each UIKit control directly the proxy object allows you to customise the appearance of a whole class of the required control.

For example, consider you want to change the UINavigationBar backgournd color so that it will be used globally by the all instances of this class. It's possible to achieve that using the next code snippet:

 

// Replace kGlobalTintColor with any color you want.
UINavigationBar.appearance().barTintColor = kGlobalTintColor

 

The navigation bar title font can be changed in the following way:

 

if let titleFont = UIFont(name: kFontName, size: 17) {
    let navigationBarTitleAttributes: [String : NSObject] =  [NSFontAttributeName: titleFont]
    UINavigationBar.appearance().titleTextAttributes = navigationBarTitleAttributes
}

Note: Replace kFontName with the name of your preferable font. You should have previously intigrated font with the project. To know more about this process refer to this link.

The other UI elements can be configured in the similar way.

For your convenience the struct with 140 color names supported by the all modern browsers is provided in Colors.swift file. You can use it to easily set colors in any place of the project like this:

let kGlobalTintColor = UIColor.sh_colorFromHexCode(Colors.CRIMSON)

Note: kGlobalTintColor is declared in the Constants.swift file. You can find there the other useful variables.

 

Per-element customization

The settings view controller can be customized in the following ways:

  • Change kSettingsControllerCompactContentWidth and kSettingsControllerRegulartContentWidth properties (declared in the Constants.swift file) to customize the settings controller size in accordance with your preferences. The changes will be applied respecting the appropriate size class automatically.
  • There are two predefined transition animations: a paper-fold and slide from the edge. The animation type can be changed in the GFEditorViewController.swift file, in viewDidLoad() method like this:
// Set the interacton controller to enable the user interactions during the animation.
settingsTransitionDelegate.interactionController = GFHorizontalSwipeInteractionController()
// Set the preferable animation type.
settingsTransitionDelegate.animationController = GFSlideAnimationController()

Note: To apply the paper-fold animation change the GFSlideAnimationController to the GFFoldAnimationController.

To simpify the configuration of the UICollectionView objects UICollectionViewHelper extenstion was created. For instance, to change the number of the collection view columns in accordance with the different size classes you can use the following method:

// MARK: - UICollectionViewDelegateFlowLayout
extension GFMainScreenViewController: UICollectionViewDelegateFlowLayout {
  func collectionView(collectionView: UICollectionView, layout collectionViewLayout: UICollectionViewLayout, sizeForItemAtIndexPath indexPath: NSIndexPath) -> CGSize {
    let currentItemSize = UICollectionViewFlowLayout.sh_itemSizeForTraitCollection(traitCollection, contentSize: collectionView.bounds.size, fixedSpacing: kGFMainScreenCollectionViewFixedSpacing, numberOfColumnsForCompactClass: 2, numberOfColumnsForRegularClass: 3)
 
    return currentItemSize
  }
}

Note: Don't set too high values of the numberOfColumnsForCompactClass and numberOfColumnsForRegularClass parameters to reduce the memory usage.

There is a set of the different helper extensions in the Helpers Xcode group. Hope you'll find them useful for your own projects.
 
Basically, it should be relatively easy to configure the most UI elements individual parameters in the Main.storyboard file. You can also find some UI-related properties in the Identity Inspector pane, in the User Defined Runtime Attributes area.

Logging

This project uses the XCGLogger to provide the advanced logging features. The logger parameters are configured in XCGLoggerHelper.swift. 
 
There are several predefined log levels that can be activated as per your preference. For example, if you pass .Debug to log.setup method the Verbose messages won't be shown and Info, Warning, Error, Severe messages will be enabled.
 
You can change the other logger parameters as depicted in the following code snippet:
 
log.setup(.Debug, showLogIdentifier: false, showFunctionName: true, showThreadName: false, showLogLevel: true, showFileNames: true, showLineNumbers: true, showDate: true, writeToFile: nil, fileLogLevel: nil)

 

Note: It is possible to set the different logger levels depending on the current build type with the help of the preprocessor macro #if DEBUG.

Troubleshooting

  • Problem:

    The code cannot be compiled because of the following errors:

      Include of non-modular header inside framework

      Error: unable to read module map contents from 'Target Support Files`

      No such module...

  • Solution:

    First of all, check the CocoaPods version that is installed on your system using the following command:

    pod --version

    It should print the version number that is equal or bigger than 1.0.1. If you have the older CocoaPods version you must upgrade it using the instructions from this link.

    If after CocoaPods upgrade you are still getting the errors try to perform the following steps: 

    First of all, run `pod install` from the terminal.
Open project in Xcode, click `Window` then `Projects`, select `GifFactoryApp` (in the right column) and click `Delete` button that is opposite to the line with the title `Derived Data`;
In the Xcode `Product` menu click `Clean`;
Build project. The contents of `DerivedData` will be recreated when the build phase finishes.

    That's it. Hope you have no problems with compilation!
